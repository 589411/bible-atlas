# 聖經像素地圖・內容風格指南與生成模板

本文件是內容品質的「憲法」。任何模型(Opus、Sonnet、Haiku)或人工新增時代資料,都必須遵守本指南。品質一致性來自規格與驗證,不依賴模型能力。

---

## 一、產品定位(寫作前必讀)

- 對象:華人/台灣讀者,常因音譯人名、地名混淆而讀不下去聖經
- 核心價值:**先給空間感與時間感,再讀故事**
- 語氣:像策略遊戲的戰報與圖鑑——精煉、有畫面感、不說教
- 立場:忠於聖經文本敘事,不加入宗派性詮釋

## 二、寫作鐵則

1. **同名消歧優先**:人物 desc 必須能讓讀者區分同名者(例:「耶羅波安(臣僕)」vs 北國王耶羅波安;施洗約翰 vs 使徒約翰)
2. **平行經文必列**:同一段歷史若有多卷書記載(撒母耳記‖歷代志、列王紀‖歷代志),ref 與 books 必須全部列出,以「;」分隔
3. **字數即紀律**:desc 60–160 字、事件 20–90 字、人物卡 8–30 字。超字數=不合格,由驗證腳本把關
4. **地名只能引用**:只能使用舞台地圖 `cities` 既有的 key。需要新地名時,先提交地圖 PR,不得在時代資料中自創座標
5. **每時代一個張力**:desc 結尾點出本時代的關鍵轉折或屬靈意義(例:分裂時代的「金牛犢成為北國的國罪」)
6. **年代標示**:統一用「約主前 XXXX 年」;有爭議的年代取通行保守值,不展開學術討論

## 三、視覺約定

- 陣營色:統一王國=紫 `united`、北國=青 `north`、南國=金 `south`、外邦帝國=`empire`、其他外邦=`foreign`
- 人物袍色呼應陣營;先知統一藍灰系;只有君王 `crown: true`
- 每時代顯示 4–8 個地名,京城列入 `capitals`

## 四、生成模板(直接複製到新對話的 System Prompt)

```
你是「聖經像素地圖」專案的時代資料編輯。你的任務是依照 era.schema.json
產出一個時代的 JSON 資料,並嚴格遵守以下規則:

【規格】
- 輸出純 JSON,不含任何說明文字或 Markdown 圍欄
- 欄位與限制以 schema 為準:desc 60–160字、事件 20–90字、
  人物 2–5 位(desc 8–30字)、事件 2–5 件、地名 4–8 個
- 地名只能從我提供的地圖 cities 清單中選用 key
- territory 用 rect [起欄,起列,迄欄,迄列] 描述,不得超出地圖範圍

【寫作風格】
- 策略遊戲圖鑑語氣:精煉、有畫面感、敘事性
- 人物 desc 必須能消除同名混淆
- 有平行記載的經文必須全部並列(以「;」分隔)
- desc 結尾點出本時代的關鍵轉折

【陣營與視覺】
- faction/tagClass:united(統一)、north(北國)、south(南國)、
  prophet(先知)、empire(帝國)、other
- 君王 crown:true;袍色呼應陣營色;先知用藍灰袍

我會提供:(1) 舞台地圖的 cities 清單 (2) 一個合格範例時代 JSON
(3) 要製作的時代名稱與涵蓋範圍。
請先輸出 JSON,我驗證後若有錯誤會回報,你只修正錯誤欄位。
```

## 五、新增時代的標準流程

1. 開新對話(Sonnet 即可),貼上「生成模板」
2. 貼上目標舞台地圖 JSON 的 `cities` 區塊
3. 貼上 `data/eras/split.json` 作為合格範例
4. 指定要製作的時代(例:「北國滅亡:主前 722 年亞述攻陷撒馬利亞」)
5. 取得 JSON → 存入 `data/eras/` → 執行 `node scripts/validate.js`
6. 驗證通過 → 加入 `data/manifest.json` → push(GitHub Action 會再驗一次)

每次生成任務的上下文 ≈ 模板 1K + 地名庫 0.5K + 範例 2K + 產出 2K,
遠低於任何模型限制。整本聖經 = 約 15 個這樣的獨立小任務。
